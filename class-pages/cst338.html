In this lab, we explored object-oriented programming concepts including inheritance and abstraction using a Monster-based class design.

We began by defining a Monster interface that declared common behaviors shared by all monsters. Concrete monster classes were then created as child classes that implemented this interface.

To further practice abstraction, we introduced abstract classes to represent monster subtypes, such as a Fire-type monster. These abstract classes provided shared functionality while leaving specific behaviors to be implemented by subclasses.

Finally, we created concrete monster implementations that extended the abstract subtype classes and fully defined all required methods. This structure demonstrated how abstraction and inheritance can be combined to create flexible, reusable, and well-organized code.
